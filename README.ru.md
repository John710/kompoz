# Kompoz

> Самостоятельный веб-редактор и визуализатор для Docker Compose.

**Kompoz** помогает редактировать, организовывать и понимать Docker Compose-проекты из единого веб-интерфейса. Поддержка нескольких проектов, линтинг YAML, интерактивные карты зависимостей и встроенный сканер сети — всё в одном лёгком контейнере.

## Быстрый старт

Самый простой способ — Docker Compose:

```bash
git clone https://github.com/John710/kompoz.git
cd kompoz
```

Отредактируйте `docker-compose.yml`, указав в `volumes` свои реальные папки с проектами, затем:

```bash
docker compose up -d
```

Откройте `http://localhost:3710`.

### Без Docker

```bash
npm install
npm start
```

## Как это работает

### Подключение проектов

Укажите в `COMPOSE_MOUNTS` одну или несколько папок на хосте:

```
COMPOSE_MOUNTS=/mnt/docker,/mnt/server
```

Kompoz автоматически определяет проекты:
- **Direct-режим** — папка с файлами `.yml` или `.env` считается одним проектом.
- **Multi-режим** — обычная папка становится каталогом, а каждая подпапка внутри неё — отдельным проектом.

Можно задать красивое имя через «трубу»:

```
COMPOSE_MOUNTS=/mnt/docker,/mnt/server,/mnt/trifi|Trifi
```

### Страницы

| Страница | Путь | Для чего |
|----------|------|----------|
| **Главная** | `/` | Выбор, создание и переключение проектов |
| **Редактор** | `/editor.html` | Редактирование compose-файлов, секретов и переменных окружения |
| **Карта контейнеров** | `/map.html` | Визуализация зависимостей сервисов — клик по карточке открывает исходный файл |
| **Карта сети** | `/homelab.html` | Сканирование LAN, поиск устройств, рисование связей между ними |

## Настройка

Задаются через переменные окружения:

| Переменная | Что делает | Пример |
|------------|------------|--------|
| `COMPOSE_MOUNTS` | Где находятся ваши проекты (через запятую) | `/mnt/docker` |
| `DATABASE_URL` | Строка подключения к PostgreSQL (опционально) | `postgres://user:pass@localhost:5432/db` |
| `AUTH_USER` / `AUTH_PASS` | Логин и пароль (опционально; включают авторизацию) | `admin` / `secret` |
| `AUTH_SECRET` | Ключ подписи cookie (случайная строка) | ` anything long ` |
| `FILE_EXT_WHITELIST` | Разрешить дополнительные расширения | `\.(yml|yaml|env|json)$` |
| `ALLOW_ALL_EXTENSIONS` | Отключить проверку расширений | `true` |
| `TZ` | Часовой пояс контейнера | `Europe/Moscow` |
| `PORT` | HTTP-порт внутри контейнера | `3710` |

## Советы

- **Сканер внутри LXC / Docker?** Для ICMP-ping сканеру нужна привилегия `NET_RAW`. В шаблоне Compose это настроено через `cap_add` и `network_mode: host`. Если устройства всё равно не находятся — убедитесь, что LXC-контейнер подключён к LAN через bridge, а не через NAT.
- **Без базы данных** — Kompoz прекрасно работает с файловой системой. PostgreSQL нужен только для пользователей и истории сканирования.
- **При первом запуске** — если проекты не видны, проверьте, что `COMPOSE_MOUNTS` указывает на правильные пути внутри контейнера.
