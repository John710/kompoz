# Kompoz

Веб-редактор Docker Compose с визуальной картой зависимостей, встроенным линтером и двумя языками интерфейса.

[🇬🇧 English version](README.md)

---

## Возможности

- **Редактор файлов** — просмотр и редактирование `docker-compose.yml`, `.env`, секретов и конфигов с подсветкой синтаксиса (CodeMirror 5).
- **Визуальная карта** — интерактивная схема сервисов, сетей и volume'ов с подсветкой связей и проброшенных портов.
- **Линтер** — проверка compose-файлов на лету через [dclint](https://www.npmjs.com/package/dclint) с переводом сообщений на русский.
- **Многопроектность** — поддержка нескольких стеков в одном интерфейсе (direct- и multi-режимы монтирования).
- **Темы** — тёмная и светлая тема, переключение без перезагрузки.
- **i18n** — русский и английский языки интерфейса. Хотите добавить свой язык? См. [Contributing](CONTRIBUTING.md).
- **Портативность** — запускается в Docker, не требует базы данных.

## Быстрый старт

```bash
git clone <репозиторий>
cd kompoz

# Отредактируй docker-compose.yml: укажи свои пути к проектам
# Затем запусти:
docker compose up -d
```

Приложение будет доступно на `http://localhost:3210`.

### Переменные окружения

| Переменная | Описание | Пример |
|------------|----------|--------|
| `COMPOSE_MOUNTS` | Список точек монтирования внутри контейнера через запятую. Можно переопределить имя проекта через `\|`. | `/mnt/docker,/mnt/server,/mnt/trifi\|Trifi` |
| `TZ` | Часовой пояс контейнера | `Asia/Vladivostok` |
| `AUTH_USER` | Логин (опционально, включает авторизацию) | `admin` |
| `AUTH_PASS` | Пароль (опционально, включает авторизацию) | `secret123` |
| `AUTH_SECRET` | Секретный ключ для подписи cookie | `random-string` |
| `COOKIE_SECURE` | Добавить флаг `Secure` в cookie (для HTTPS) | `true` |
| `FILE_EXT_WHITELIST` | Пользовательское регулярное выражение для разрешённых расширений | `\.(yml\|yaml\|env)$` |
| `ALLOW_ALL_EXTENSIONS` | Отключить проверку расширений файлов | `true` |

### Режимы монтирования

- **Direct-mode** — если в папке есть `.env`, `compose/` или `.yml` файлы, она считается одним проектом. Имя берётся из `basename` пути (или из `\|NAME`).
- **Multi-mode** — если в папке нет признаков проекта, каждая подпапка внутри неё становится отдельным проектом.

## Технологии

- **Backend:** Node.js 22, Express 5
- **Frontend:** Vanilla JS, CodeMirror 5, D3 (zoom/pan на карте)
- **Linting:** dclint, yaml
- **Deploy:** Docker Compose
- **CI:** GitHub Actions (авто-релиз + Docker-образ)

## Безопасность

- **Авторизация:** установите `AUTH_USER` и `AUTH_PASS`, чтобы включить защиту по логину и паролю. Все страницы и API требуют аутентификации, если обе переменные заданы.
- **Защита от timing-атак:** логин и проверка токенов используют `crypto.timingSafeEqual`.
- **Rate limiting:** на логин действует ограничение — 5 попыток на 15 минут с одного IP.
- **Защита от path traversal:** `safeResolvePath` использует `fs.realpathSync` для блокировки symlink за пределами корня монтирования.
- **XSS:** фронтенд рендерит динамический контент через DOM API и экранирование HTML-entities вместо `innerHTML`.
- **CSP-ready:** inline обработчики событий мигрированы на `addEventListener`.
- **Whitelist расширений:** при создании и сохранении файлов проверяется расширение по настраиваемому списку.
